# rfuji web app

A browser FAIR-assessment client for [rfuji](https://github.com/choxos/rfuji).
React + TypeScript + Vite + Tailwind. It reimplements the scoring logic in
TypeScript and runs entirely client-side.

## How it differs from the R engine

The R package (`rfuji::assess_fair()`) harvests landing pages, content-negotiated
RDF/XML, and signposting links. A browser cannot fetch those cross-origin
(CORS), so this app uses only **CORS-enabled registry APIs** (DataCite,
Crossref). As a result some metrics (notably F4 searchable, and data-link based
tests) score lower here than in the R engine. The scores still agree with the R
engine for everything derivable from registry metadata.

Reference data (metric definitions, SPDX licenses, file formats, access rights,
protocols) is the **single source of truth** generated by the R package's
`data-raw/` pipeline; `npm run sync-data` copies `inst/extdata/web/*.json` into
`public/data/`.

## Develop

```sh
npm install
npm run sync-data   # copy reference JSON from ../inst/extdata/web
npm run dev         # http://localhost:5173
npm run build       # production build (base /rfuji/app/)
npm run typecheck
```

## Deploy

`.github/workflows/deploy-app.yaml` builds this app and publishes `dist/` to the
`gh-pages` branch under `/app`, so it is served at
`https://choxos.github.io/rfuji/app/` alongside the pkgdown docs at the root.
